How a client uses NotifyHub.

A client registers once, sends canonical requests, and lets NotifyHub own routing, providers, callbacks, and retries.

1. Register the client

Create a client identity and receive an API key. The client uses that key on every request.

POST /v1/clients
{
  "tenant_id": "tenant-a",
  "client_name": "billing-service",
  "allowed_channels": ["email", "sms", "whatsapp", "push", "webhook"]
}

2. Send a canonical request

Put the event, template key, channel list, recipient, and variables in one request. Do not include provider credentials.

POST /v1/notification-requests
Authorization: Bearer <api_key>

3. Read lifecycle updates

NotifyHub sends normalized delivery lifecycle events back to your system so you can update your own records.

Home · Docs · Architecture

Built like a control plane, not a handoff script.

Requests enter once, get normalized, and move through policy, templates, routing, dispatch, callbacks, and observability.

Ingress

API + Kafka

The API validates and persists requests, then hands delivery orchestration to the worker through Kafka.

Execution

Worker + Connectors

The worker resolves templates, preferences, and bindings before dispatching through the relevant connector.

Return path

Callbacks

Provider delivery updates and inbound replies come back through the callback gateway and become durable events.

Operations

Metrics and logs

Prometheus, Grafana, Kafka UI, and Adminer sit beside the runtime so teams can debug with real signals.

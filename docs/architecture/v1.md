# V1 Architecture Notes

The initial scaffold establishes the repository, process boundaries, and local runtime shape for:

- `api`
- `worker`
- `callback-gateway`
- `connector-email`
- `connector-sms`
- `connector-webhook`

The next iteration should add:

- persistent request storage in PostgreSQL
- Kafka-backed ingestion and worker consumption
- delivery attempt state machine
- provider registry and routing policy evaluation
